# Project Spec

> Replace this file with your project specification.

SwarmBoard's PM agent reads this file to create and groom tickets. Write your project requirements here in markdown — user stories, features, architecture decisions, constraints, and acceptance criteria.

## Example Structure

```markdown
# My Project

## Overview
What the project does and why.

## Features
### Feature 1 — Title
- User story: As a [role], I want [capability] so that [benefit]
- Acceptance criteria:
  - [ ] Criterion 1
  - [ ] Criterion 2

### Feature 2 — Title
...

## Technical Constraints
- Language/framework requirements
- Performance targets
- Security requirements

## Non-Functional Requirements
- Deployment targets
- Browser/platform support
```

## Tips

- Be specific about acceptance criteria — the test agent uses them to verify work
- Include technical constraints so the dev agent makes the right choices
- Mark priorities (P0/P1/P2) so the PM agent can prioritize the backlog
